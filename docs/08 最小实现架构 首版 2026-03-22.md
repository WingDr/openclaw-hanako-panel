# OpenClaw Panel 最小实现架构

更新日期 2026-03-22

这份文档专门收束 OpenClaw Panel 的最小实现架构。目标是把前面的方案明显缩小，只保留最主要的功能，先做一个可以稳定跑起来、方便继续扩展的首版实现。 

## 一句话结论

先不要做完整 panel。先做一个最小的 web 面板，只实现三件事。

- 分 agent 的 chat
- 实时 log 监控
- agent 和渠道状态监控

技术上先收缩成：

- 前端 `React + TypeScript + Vite + React Router + Zustand + TailwindCSS`
- 中间层 `panel proxy`
- 后端仍然是 `OpenClaw Gateway`

先不做复杂主题系统。先不做 config、cron、skills、文件管理。先把最主要的使用闭环做出来。

## 一、首版的明确边界

## 只做什么

### 1. 分 agent 的 chat

前端需要支持：

- 选择 agent
- 查看该 agent 下的 session 列表
- 新建独立 session
- 进入指定 session 聊天
- 查看流式回复与基础 tool stream

### 2. 实时 log 监控

前端需要支持：

- 查看 Gateway 日志
- 自动滚动
- 暂停滚动
- 简单搜索
- 简单 level 过滤

### 3. agent 和渠道状态监控

前端需要支持：

- Gateway 连接状态
- agent 状态
- channel 状态
- 最近 session 活动概览

## 明确先不做什么

为了把体量真正压下来，这一版先不做：

- config 编辑
- cron 管理
- skills 管理
- models 管理
- 文件管理
- 多用户权限
- session pin / tag / archive
- 命令面板
- split screen
- 复杂右侧上下文侧栏
- 设计 token 系统
- 大量宿主机运维接管

## 二、最小技术栈

## 前端

建议最小前端组合：

- React
- TypeScript
- Vite
- React Router
- Zustand
- TailwindCSS

## 为什么先不要上 TanStack Query

因为这一版页面少，功能边界很窄。HTTP 数据获取量也不大。先自己封装一个轻量 `api.ts` 就够了。等页面和接口都稳定之后，如果状态同步复杂度真的上来，再补 TanStack Query。

## 日志组件建议

第一版先不要接完整的大日志组件库。先做一个简单日志面板。

只要支持：

- 滚动
- follow
- filter
- search
- buffer 裁剪

如果后面发现性能不够，再切到 `@melloware/react-logviewer`。

## panel proxy

建议保持：

- Node.js
- Fastify
- ​`@fastify/websocket`

第一版甚至可以不落数据库。若只是保存很少量本地状态，先用内存或简单 JSON 文件都够。

## 四、最小数据流

这一版的数据流只保留三条主线。

## 1. chat 数据流

浏览器通过 proxy 发聊天请求。 proxy 转发给 OpenClaw Gateway。 流式事件再由 proxy 推回浏览器。

### 浏览器到 proxy

- ​`chat.send`
- ​`chat.abort`
- ​`session.create`
- ​`session.open`

### proxy 到浏览器

- ​`chat.started`
- ​`chat.delta`
- ​`chat.done`
- ​`chat.error`

## 2. logs 数据流

proxy 内部轮询 `logs.tail`。 浏览器只接收日志追加事件。

### 浏览器到 proxy

- ​`logs.subscribe`
- ​`logs.unsubscribe`

### proxy 到浏览器

- ​`logs.append`
- ​`logs.reset`

## 3. status 数据流

状态页不要求每个指标都强实时。可以分成两层。

### 准实时

- agent 状态
- channel 状态
- recent sessions

方式：

- 页面首次通过 HTTP 拉取
- 之后每 5 到 10 秒刷新一次
- 或由 proxy 定时推送 `status.snapshot`

### 实时

- gateway 连接状态通过 WebSocket 推送 `system.connection`
